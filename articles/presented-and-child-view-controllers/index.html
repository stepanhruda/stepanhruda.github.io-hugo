Part 4: Presented and child view controllers

Putting the easy into easy transitioning

Stepan Hruda on 04 June 2014

This article is part of the Transition with STPTransitions series.

So far, we have only worked with transitions inside a UINavigationController. iOS has two additional ways of switching between view controllers, though: presentation via presentViewController:animated:completion: (with dismissal via dismissViewController:animated:completion:), and containment, where the driver is transitionFromViewController:toViewController:animations.

Good news, everyone: any STPTransition you write works for all three types of transitions. In other words, you can reuse them in any of the provided methods.

Presentation

STPTransitions provides methods with signatures corresponding to UIKit's. We can make the example from the previous article on interactivity work with one duh change:

  • instead of push and pop methods, we use present and dismiss
- (void)setUpForwardTransition:(STPTransition *)transition {
    ...

    __weak __typeof(self) weakSelf = self;
    transition.onGestureTriggered = ^(STPTransition *transition, UIGestureRecognizer *recognizer) {
        [weakSelf presentViewController:[STPSecondViewController new]
                        usingTransition:transition
                           onCompletion:nil];
//      [self.navigationController pushViewController:[STPSecondViewController new]
//                                    usingTransition:transition];
    };
    ...
}

- (void)setUpReverseTransition:(STPTransition *)transition {
    __weak __typeof(self) weakSelf = self;
    transition.onGestureTriggered = ^(STPTransition *transition, UIGestureRecognizer *recognizer) {
        [weakSelf dismissViewControllerAnimated:YES completion:nil];
//      [weakSelf popViewControllerAnimated:YES];
    };
    ...
}
  • (also, we can completely get rid of the navigation controller, it's not needed for this setup)
Ta-dah! The exact same transition using a completely different mechanism.
